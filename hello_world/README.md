## Используемые технологии:

**Backend:**

- `Python 3` — основной язык разработки бэкенда и ML-модуля.
- `FastAPI` — асинхронный веб-фреймворк для создания API.
- `TensorFlow / Keras` — библиотека для создания и использования сверточной нейросети (CNN).
- `OpenCV` — библиотека для предварительной обработки изображений (рисунков пользователя).
- `NumPy` — работа с матрицами данных.

**Frontend:**

- `TypeScript` - для создания пользовательского интерфейса.

**Database:**

- `MongoDB` — основная база данных для хранения информации об иероглифах.
- `PyMongo` — асинхронный драйвер для работы с MongoDB из Python.

## Видение проекта:

### Пользовательские роли

Распознавание: Пользователь рисует иероглиф на холсте, система распознает его и предлагает наиболее вероятные варианты.

Изучение: Просмотр карточки иероглифа: значения, чтения (онные/кунные), количество черт, визуализация порядка написания (Stroke Order), декомпозиция

Поиск: Поиск кандзи по количеству черт, радикалам, году изучения JLPT.

### Коллекции


основаны на данных
+ [kanji-stroke-order](https://github.com/rouleaup88/Kanji-stroke-order) - Данные `kanjiVG_data` (координаты SVG) для отрисовки анимации написания. 
+ [kanjidic](https://github.com/nramkissoon/Kanjidicparser) - Основные метаданные: чтения (`On/Kun`), значения (`meanings`), количество черт и уровень JLPT.
+ [radkfile/kradfile](https://www.edrdg.org/krad/kradinf.html) -  Список радикалов (составных частей) для каждого иероглифа.
+ [tatoeba](https://github.com/Tatoeba/tatoeba2) - Популярные слова и словосочетания, в которых встречается этот иероглиф.
+ [JMDict](https://github.com/scriptin/jmdict-simplified) - Примеры целых предложений с переводом для демонстрации контекста.


## Коллекция 1: `kanji` (Ядро системы)

Здесь хранится «паспорт» иероглифа.

| Поле         | Тип              | Описание                                      |
|--------------|------------------|-----------------------------------------------|
| `_id`        | ObjectID         | Уникальный идентификатор документа            |
| `char`       | String           | Сам иероглиф (например, "語")                 |
| `unicode`    | String           | Код символа (например, "8a9e")                |
| `strokes`    | Int32            | Количество черт                               |
| `jlpt_level` | Int32 \| null    | Уровень JLPT (1–5), null если не входит       |
| `meanings`   | Array\<String\>  | Массив значений (["language", "word"])        |
| `readings`   | Object           | Объект с полями `on: Array<String>` и `kun: Array<String>` |
| `kanjiVG_data` | String         | SVG-пути для анимации порядка черт            |
| `radical_ids`| Array\<ObjectID\>| Ссылки на документы коллекции `radicals`      |

---

## Коллекция 2: `radicals` (Справочник компонентов)

Нужна для функции «Поиск по радикалам».

| Поле      | Тип      | Описание                                      |
|-----------|----------|-----------------------------------------------|
| `_id`     | ObjectID | Уникальный идентификатор документа            |
| `char`    | String   | Символ радикала (например, "言")              |
| `meaning` | String   | Что означает этот компонент (например, "speak") |

---

## Коллекция 3: `vocabulary` (Слова из JMDict)

Слова, состоящие из этого иероглифа.

| Поле        | Тип      | Описание                                          |
|-------------|----------|---------------------------------------------------|
| `_id`       | ObjectID | Уникальный идентификатор документа                |
| `kanji_id`  | ObjectID | Ссылка на иероглиф из коллекции `kanji`           |
| `word`      | String   | Слово целиком (например, "日本語")                |
| `reading`   | String   | Как читается слово (хирагана/ромадзи)             |
| `translation` | String | Перевод слова                                     |

---

## Коллекция 4: `sentences` (Примеры из Tatoeba)

Целые фразы для продвинутого изучения.

| Поле            | Тип               | Описание                                        |
|-----------------|-------------------|-------------------------------------------------|
| `_id`           | ObjectID          | Уникальный идентификатор документа              |
| `vocabulary_id` | ObjectID \| null  | Ссылка на конкретное слово (опционально)        |
| `kanji_id`      | ObjectID          | Ссылка на основной иероглиф                     |
| `japanese`      | String            | Предложение на японском                         |
| `russian`       | String            | Перевод предложения                             |
